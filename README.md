# Diana Artist Portfolio

Предрелизная версия авторского сайта-портфолио художницы Дианы.

Сайт собран как живой студийный архив: работы, дневниковые записи, процесс, наличие картин и контактный сценарий объединены в одну статическую страницу. Визуальная система опирается на эстетику бумажного арт-зина: тёплая фактура, редакционные линии, скотч, рукописные пометки, асимметричные коллажи и реальные исходные изображения.

## Что внутри

- `site/` — готовая папка для деплоя.
- `site/index.html` — главная страница.
- `site/src/` — JavaScript, данные и стили сайта.
- `site/public/assets/` — изображения, портреты, архивные фото и видео.
- `.github/workflows/deploy-pages.yml` — workflow для деплоя на GitHub Pages.

Исходные макеты, экспорт чата, рабочие бэкапы и промежуточные проверки намеренно не включаются в git через `.gitignore`.

## Возможности сайта

- Hero-секция с коллажем работ, портретом и process video.
- Раздел о Диане и манифест проекта.
- Каталог из 10 реальных работ.
- Поиск, фильтрация по категориям, сортировка и переключение grid/list.
- Детальный overlay работы с историей, метаданными, тегами и видео процесса.
- Архив дневниковых записей с просмотром изображений.
- Раздел работ в наличии.
- Сценарий покупки через предзаполнение контактной формы.
- Генерация текста сообщения для Telegram без backend-отправки.
- Адаптация под desktop и mobile.
- Базовая доступность: semantic HTML, focus states, Escape close, visible controls.

## Локальный запуск

Из корня репозитория:

```bash
cd /workspaces/New-design-site
python3 -m http.server 4173 -d site
```

Открыть:

```text
http://localhost:4173
```

Остановить сервер:

```text
Ctrl + C
```

Если порт занят:

```bash
python3 -m http.server 3000 -d site
```

## Открыть на телефоне

Телефон и компьютер должны быть в одной Wi-Fi сети.

Узнать локальный IP компьютера:

```bash
hostname -I
```

Обычно нужен первый адрес из вывода, например `192.168.1.23`.

Запустить сервер:

```bash
python3 -m http.server 4173 -d site --bind 0.0.0.0
```

Открыть на телефоне:

```text
http://192.168.1.23:4173
```

Если не открывается, чаще всего причина в VPN, firewall, разных Wi-Fi сетях или неверном IP.

## Деплой на GitHub Pages

1. Создать репозиторий на GitHub.
2. Запушить текущий проект в ветку `main`.
3. В GitHub открыть `Settings` -> `Pages`.
4. В `Build and deployment` выбрать `GitHub Actions`.
5. Workflow `Deploy static site to GitHub Pages` сам задеплоит содержимое папки `site/`.

После успешного запуска Actions сайт будет доступен по адресу GitHub Pages, который появится в summary workflow.

## Предрелизные риски

- Сайт статический и не отправляет форму на backend. Он только готовит текст для Telegram.
- Часть цен и размеров намеренно не выдумывалась. Если канонические данные появятся позже, их нужно обновить в `site/src/data/artworks.js`.
- Видео лежат прямо в репозитории. Сейчас размер приемлемый для предрелиза, но при росте медиатеки лучше подключить CDN или отдельное хранилище.
- Используются системные fallback-шрифты, близкие к макету. Для более точного pixel-perfect результата стоит добавить лицензированные webfont-файлы.

## Проверка перед публикацией

```bash
python3 -m http.server 4173 -d site
```

Проверить в браузере:

- `http://localhost:4173`
- desktop ширина около `1440px`;
- mobile ширина около `390px`;
- поиск по каталогу;
- открытие карточки работы;
- кнопка `Уточнить покупку`;
- генерация Telegram-сообщения.
